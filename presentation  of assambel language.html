📖 Introduction

We built an encryption/decryption system in 8086 assembly using XOR operation. The user types a message, it's encrypted with key 55, shows gibberish, then decrypts back.

Ciphertext = Plaintext XOR Key
Plaintext = Ciphertext XOR Key

Why XOR? No separate decrypt routine needed.

⚠️ Problem Statement

  • Accept string (max 50 chars).
  • Encrypt using XOR with a byte key.
  • Show encrypted text (gibberish).
  • Decrypt using same routine → original.
  • Single function for both enc/dec.

📚 Literature View

The XOR cipher (Vernam cipher) is one of the oldest symmetric ciphers. Modern stream ciphers like RC4, Salsa20 still use XOR with a pseudo‑random key stream.

Mathematical property: (P ⊕ K) ⊕ K = P

Our 8086 implementation shows this at the lowest hardware level.

🧠 XOR Explained (Example)

Take character 'A' with Key = 55.

'A' ASCII 65  → 01000001
Key 55        → 00110111
XOR           → 01110110 = 118 → 'v'  (ciphertext)

Decrypt: 118 → 01110110
XOR 55       → 00110111
Result       → 01000001 = 65 → 'A'

Same operation works for any character, number, or symbol.

💻 Code Walkthrough (Hover on any underlined text)

include 'emu8086.inc'

.MODEL SMALL
.STACK 100H

.DATA
    KEY DB 55
    buffer DB 51 DUP(0)
    bufLen DW 0

.CODE
MAIN PROC
    MOV AX, @DATA
    MOV DS, AX

    PRINT '--- ADIL & BILAL ---'
    PRINTN
    
    LEA DX, msg_prompt
    MOV AH, 09H
    INT 21H
    
    LEA DI, buffer
    MOV DX, 50
    CALL GET_STRING
    
    CALL GetLength
    MOV bufLen, AX

    MOV AH, 09H
    MOV DX, OFFSET sOriginal
    INT 21H
    CALL DisplayBuffer

    CALL TranslateBuffer
    
    MOV AH, 09H
    MOV DX, OFFSET sEncrypt
    INT 21H
    CALL DisplayBuffer

    CALL TranslateBuffer
    
    MOV AH, 09H
    MOV DX, OFFSET sDecrypt
    INT 21H
    CALL DisplayBuffer

    MOV AH, 4CH
    INT 21H
MAIN ENDP

TranslateBuffer PROC
    MOV CX, bufLen
    CMP CX, 0
    JE  DoneXor
    MOV SI, 0
L1:
    MOV AL, buffer[SI]
    XOR AL, KEY
    MOV buffer[SI], AL
    INC SI
    LOOP L1
DoneXor:
    RET
TranslateBuffer ENDP

DisplayBuffer PROC
    MOV CX, bufLen
    CMP CX, 0
    JE  DoneDisp
    MOV SI, 0
L2:
    MOV DL, buffer[SI]
    MOV AH, 02H      ; print char
    INT 21H
    INC SI
    LOOP L2
DoneDisp:
    RET
DisplayBuffer ENDP

GetLength PROC
    LEA SI, buffer
    MOV AX, 0
CountLoop:
    CMP BYTE PTR [SI], 0   ; null check
    JE  Finished
    INC AX                 ; length++
    INC SI                 ; next char
    JMP CountLoop          ; repeat
Finished:
    RET
GetLength ENDP

DEFINE_GET_STRING
END MAIN

🔁 Loops & Conditions Used

  • LOOP – uses CX as counter. Automatically decrements and jumps.
  • CMP + JE – compare and jump if equal (used for null check, zero length).
  • JMP – used in GetLength to keep looping until null terminator.
MOV CX, bufLen   ; load counter
CMP CX, 0        ; check if zero
JE  DoneXor      ; if zero, skip loop
MOV SI, 0
L1: ... LOOP L1  ; decrement CX, jump to L1 if CX>0

📟 Register Roles

  • AX – DOS function numbers & return values.
  • AL – stores current character for XOR.
  • AH – DOS function selector.
  • CX – loop counter (string length).
  • SI – index into buffer (source).
  • DI – destination for GET_STRING.
  • DX – string addresses or max length.
  • DS – points to data segment.
