🧠 Slide 4 – XOR Kya Hai? (Simple)
Adil:
"Pehle samajhte hain XOR kya hai.
XOR do bits ko compare karta hai.
Agar dono bits same hain to result 0 hota hai.
Agar dono different hain to result 1 hota hai.
Example:
0 XOR 0 = 0
0 XOR 1 = 1
1 XOR 0 = 1
1 XOR 1 = 0"
Bilal:
"Computer ke andar sab kuch binary (0 aur 1) mein store hota hai.
Har character ka ek ASCII code hota hai.
Jaise 'A' ka ASCII 65 hai, jo binary mein 01000001 hai.
Humne key rakhi hai 55 – binary mein 00110111.
Jab hum 01000001 ko XOR karte hain 00110111 se, to milta hai 01110110 jo 118 hai – aur ye character hai 'v'.
Toh 'A' encrypt ho kar 'v' ban gaya.
Ab agar hum 'v' (118) ko wapas XOR karein 55 se, to wapas 01000001 (65) aata hai – yani 'A' wapas.
Yahi XOR ka magic hai."
🔀 Slide 5 – Gibberish Kahan Se Aata Hai?
Adil:
"Gibberish isliye dikhta hai kyunki XOR karne ke baad ASCII values badal jaati hain.
Kuch new ASCII values non‑printable hoti hain, kuch alag symbols ban jaate hain.
Is liye aapko ajeeb characters dikhte hain."
Bilal:
"Agar aap pura sentence encrypt karein, to har character badal jaata hai.
Aur result aisa lagta hai jaise koi alien language ho.
Phir jab aap wapas XOR karein, to sab original wapas aa jaata hai."
📊 Slide 6 – Flowchart (Methodology)
Adil:
"Ab main flowchart samjhata hoon.
1. Program start hota hai.
2. Data segment initialize hota hai.
3. Screen par title aur prompt print hota hai.
4. GET_STRING macro user se input leti hai aur buffer mein store karti hai.
5. GetLength procedure buffer ki length count karti hai (null terminator tak).
6. Original message print hota hai.
7. Pehli baar TranslateBuffer call hota hai – ye XOR karta hai → encryption.
8. Ciphertext (gibberish) print hota hai.
9. Doosri baar TranslateBuffer call hota hai – XOR again → decryption.
10. Decrypted original message print hota hai.
11. Program exit hota hai."
Bilal:
"Dhyaan dein: TranslateBuffer ek hi baar likha gaya hai, lekin use do baar call kiya gaya.
Pehli baar encrypt, doosri baar decrypt.
Isi ko kehte hain symmetric encryption."
💻 Slide 7 – Code Main Parts
Adil:
"Data segment mein humne KEY DB 55 rakha hai – ye hamara key hai.
buffer DB 51 DUP(0) – isme user ka input store hoga.
bufLen DW 0 – yahan string ki length store hogi."
Bilal:
"Main procedure mein:
MOV AX, @DATA aur MOV DS, AX – data segment set karta hai.
PRINT macro se title print hota hai.
LEA DX, msg_prompt – prompt ka address DX mein.
MOV AH, 09H – DOS function 09h (print string).
INT 21H – DOS interrupt call.
LEA DI, buffer – buffer ka address DI mein.
MOV DX, 50 – maximum 50 characters.
CALL GET_STRING – input leta hai.
CALL GetLength – length calculate karta hai.
MOV bufLen, AX – length store karta hai.
Phir CALL TranslateBuffer (encrypt), ciphertext print, dubara CALL TranslateBuffer (decrypt), decrypted print.
MOV AH, 4CH aur INT 21H – exit."
⚙️ Slide 8 – TranslateBuffer (XOR Logic)
Bilal:
"TranslateBuffer procedure:
MOV CX, bufLen – CX mein length daalo.
CMP CX, 0 – agar length zero hai to jump DoneXor.
MOV SI, 0 – SI zero (pehle character).
L1: MOV AL, buffer[SI] – ek character AL mein laao.
XOR AL, KEY – AL ko key ke saath XOR karo.
MOV buffer[SI], AL – result wapas buffer mein store karo.
INC SI – agle character par jao.
LOOP L1 – CX kam karo, agar CX zero nahi to L1 par jao.
RET – wapas jao."
Adil:
"Yeh procedure dono baar same kaam karta hai – pehle encrypt, phir decrypt."
🔄 Slide 9 – Loops & Conditions
Bilal:
"Humne teen tarah ke loops aur conditions use kiye hain:
1. LOOP – hardware loop. CX counter hota hai, apne aap CX kam karta hai aur jab tak zero nahi hota, jump karta hai.
2. CMP + JE – compare karo, agar equal hai to jump. Zero length check karne ke liye.
3. JMP – unconditional jump. GetLength mein null terminator tak loop karne ke liye."
Adil:
"Example: CMP BYTE PTR [SI], 0 – check karta hai ke kya current byte null (0) hai."
📟 Slide 10 – Register Roles
Bilal:
"AX – DOS function numbers aur return values.
AL – current character XOR karne ke liye.
CX – loop counter (bufLen).
SI – source index, buffer mein current position.
DI – destination index (GET_STRING use karta hai).
DX – string addresses aur max length.
DS – data segment, .DATA variables ko point karta hai."
Adil:
"Assembly mein registers ka sahi istemal bahut important hai."
